# LiqPay та платіжні логотипи

Ця стаття описує технічну реалізацію блоку оплати через LiqPay та адаптивної сітки логотипів довіри (Visa, Mastercard, Apple Pay тощо).

## Адаптивна сітка логотипів

Для забезпечення коректного відображення на мобільних пристроях (320px - 375px) без виходу за межі контейнера, використовується наступний підхід:

- **Контейнер**: `display: flex`, `justify-content: center`.
- **Відступи**: `gap: clamp(2px, 1.2vw, 8px)`. Це дозволяє відступам динамічно зменшуватися на маленьких екранах.
- **Елементи (Лого)**:
    - `flex: 1 1 40px` — дозволяє елементам рівномірно стискатися.
    - `maxWidth: 54px` — обмежує ріст до стандартного розміру.
    - `aspectRatio: '54/34'` — підтримує правильні пропорції логотипа при зміні ширини.

## Процес оплати

1. **Серверна частина**: `src/app/api/payment/prepare-liqpay/route.js` формує `data` та `signature` для LiqPay API.
2. **Клієнтська частина**: `src/app/checkout/page.jsx` створює динамічну форму та перенаправляє користувача на сторінку оплати LiqPay.
3. **Після оплати**: LiqPay перенаправляє користувача назад на `src/app/payment/success` або `failure`.

## Пов'язані файли
- `src/app/checkout/page.jsx` — рендеринг сітки та ініціація платежу.
- `logo-liqpay-symbol.svg` — іконка для селектора методів оплати.
