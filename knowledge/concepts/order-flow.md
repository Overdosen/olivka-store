# Процес оформлення замовлення (Order Flow)

Цей концепт описує повний життєвий цикл замовлення в Store Olivka, від додавання в кошик до обробки сервером.

## Потоки даних
1. **Кошик**: Управління через `CartContext.js`. Перевірка `stock` та розрахунок безкоштовної доставки (від 2500 грн).
2. **Checkout**: Збір даних про клієнта, методів оплати та [[concepts/delivery|доставки]].
3. **Обробка (Submit)**:
   - Створення запису в [[concepts/database-schema|таблиці orders]].
   - Тригер вебхука n8n для CRM.
   - Очищення кошика.

## Гостьове замовлення (Guest Checkout)
Система дозволяє оформлювати замовлення без обов'язкової реєстрації. 
- **Поведінка**: `AuthModal` пропонує вхід/реєстрацію, але його закриття не блокує доступ до чекауту.
- **Редиректи**: 
    - Авторизовані користувачі після замовлення перенаправляються в `/account`.
    - Гості (авторизація необов'язкова) перенаправляються на сторінку успіху `/payment/success` з параметром `order_id`.

## Технічні компоненти
- `src/app/checkout/page.jsx` — Основна логіка оформлення.
- `src/app/account/` — Перегляд історії замовлень (тільки для авторизованих).
- `src/app/payment/success/` — Сторінка підтвердження для всіх типів замовлень.
