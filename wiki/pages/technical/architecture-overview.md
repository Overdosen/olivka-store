# Архітектура проекту Store Olivka

Проект побудований на сучасному стеку технологій з акцентом на продуктивність та простоту підтримки.

## Технологічний стек
- **Frontend**: Next.js 15 (App Router)
- **Мова**: JavaScript (Babel/ESNext)
- **Стилізація**: Tailwind CSS 4.0
- **База даних**: Supabase (PostgreSQL)
- **Анімації**: Framer Motion
- **Стан**: React Context API
- **Хостинг**: Vercel

## Структура проекту

### 📂 `src/app/`
Використовує **Next.js App Router**. Основні маршрути:
- `/` — Головна сторінка.
- `/catalog/` — Каталог товарів.
- `/product/[id]/` — Картка товару.
- `/checkout/` — Оформлення замовлення.
- `/admin/` — Панель управління магазином.
- `/account/` — Особистий кабінет клієнта.

### 📂 `src/components/`
- **Global**: `Header.jsx`, `Footer.jsx`, `CartDrawer.jsx`.
- **Selectors**: `NovaPoshtaSelector.jsx`, `UkrPoshtaSelector.jsx`.
- **UI**: Загальні компоненти інтерфейсу (модалки, кнопки).

### 📂 `src/context/`
- `CartContext.js`: Керує кошиком, кількістю товарів та залишкам на складі.
- `AuthContext.js`: Керує авторизацією через Supabase Auth.

## Робочий процес даних
1. **Fetching**: Дані про товари завантажуються безпосередньо з Supabase на клієнтських або серверних компонентах.
2. **Checkout**: При підтвердженні замовлення дані записуються в таблицю `orders` у Supabase та паралельно відправляються на вебхук (n8n).
3. **Admin**: Адмін-панель використовує RLS (Row Level Security) для захисту даних клієнтів.

## Робота з ШІ-агентами
Проект містить папку `.agents/`, де зберігаються правила та скіли для ШІ-агентів, що допомагають у розробці.
