# Логістика та Доставка

Проект Store Olivka інтегрований з двома основними поштовими операторами України: **Нова Пошта** та **Укрпошта**.

## Нова Пошта (Nova Poshta)
Інтеграція реалізована через компонент [NovaPoshtaSelector.jsx](file:///d:/Overdose/%D0%92%D0%B0%D0%B6%D0%BD%D0%BE/Olivka_store/src/components/NovaPoshtaSelector.jsx).

### Методи доставки
1. **У відділення / Поштомат**:
    - Використовує пошук за назвою міста.
    - Отримує список відділень для обраного міста.
    - **Формат адреси**: `НП Відділення: [Місто], [Відділення]`.
2. **Кур'єром**:
    - Користувач вводить адресу вручну.
    - **Формат адреси**: `НП Кур'єр: [Місто], [Вулиця], [Будинок]`.

### Технічні деталі
- Використовується прямий запит до JSON API Нової Пошти з клієнта.
- Ключ API береться з `NEXT_PUBLIC_NOVA_POSHTA_API_KEY`.

## Укрпошта
Інтеграція реалізована через компонент [UkrPoshtaSelector.jsx](file:///d:/Overdose/%D0%92%D0%B0%D0%B6%D0%BD%D0%BE/Olivka_store/src/components/UkrPoshtaSelector.jsx).

### Провайдер даних
Оскільки API Укрпошти має обмеження по CORS, ми використовуємо внутрішній **Proxy Route**:
- [src/app/api/ukrposhta/route.js](file:///d:/Overdose/%D0%92%D0%B0%D0%B6%D0%BD%D0%BE/Olivka_store/src/app/api/ukrposhta/route.js).

### Робота селектора
1. Пошук населеного пункту за назвою.
2. Пошук поштового індексу (ВПЗ).
3. **Формат адреси**: `Укрпошта: [Індекс], [Область], [Район], [Населений пункт], [Відділення]`.

---
## Важливо (Гігієна адрес)
- Всі адреси зберігаються у єдиному рядку в колонці `address` таблиці `orders`.
- Ці дані автоматично відображаються в адмін-панелі та кабінеті користувача.
