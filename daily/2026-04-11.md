### Session (14:00–20:00) - Checkout Payment UI — LiqPay & Trust Badges

**Context:** Доопрацювання блоку вибору методу оплати на сторінці `/checkout`. Мета — замінити текстові лейбли на справжні логотипи платіжних систем та вирівняти їх у один акуратний ряд.

**Key Exchanges:**
- Замінили іконку картки та текст `"карткою"` → `"Банківською карткою"` + символ LiqPay з `/logo-liqpay-symbol.svg`.
- Отримали логотипи платіжних систем з репозиторію `github.com/datatrans/payment-logos`: `apple-pay.svg`, `google-pay-alt.svg`, `visa.svg`, `mastercard.svg`.
- Вбудували SVG-шляхи прямо в JSX як inline-компоненти (щоб уникнути зайніх HTTP-запитів та CORS).
- Додали PrivatPay (`/24-pay-mark_border.svg`) між Mastercard і Apple Pay.
- Виправлено серію синтаксичних помилок JSX: незакриті теги, regexp literal errors від inline SVG-шляхів.
- Виправлено некоректні DOM-атрибути: `stroke-width` → `strokeWidth`, `fill-rule` → `fillRule`, `clip-rule` → `clipRule`.
- Уніфіковано розміри всіх іконок до `54×34px`.
- Фінальне виправлення: контейнер іконок переведено з `flex-start + gap: 0.5rem` на `space-between + width: 100%`.
- PrivatPay img: замість фіксованих `width=42 height=26` → `style={{ width: '100%', height: '100%', objectFit: 'contain' }}`.

**Decisions Made:**
- Inline SVG замість `<img src>` для зовнішніх логотипів — надійніше та швидше.
- Фіксована висота картки `34px`, ширина `54px` — оптимально для мобільного вигляду.
- `space-between` без фіксованого gap — іконки самі заповнюють простір незалежно від ширини екрана.
- PrivatPay рендериться через `<img>` зі стилем `objectFit: contain` — SVG без хардкоду розмірів.

**Lessons Learned:**
- Inline SVG в JSX вимагає camelCase атрибути — `strokeWidth`, `fillRule`, `clipRule`. Іноді SVG з веб містить дефісні атрибути, які треба конвертувати вручну.
- Великі SVG path-рядки ламають JSX-парсер якщо містять специфічні символи. Треба перевіряти через `npm run dev` одразу після вставки.
- `flex-start` виглядає охайно тільки якщо іконок достатньо багато. При 5+ іконках `space-between` дає кращий вигляд.
- При роботі зі `replace_file_content` TargetContent повинен збігатися побайтово з файлом — навіть пробіли мають значення.

**Files Changed:**
